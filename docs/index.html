1. ¿Qué es y para qué sirve?1. What is it and what is it for?

El conector SQL Server permite que el agente de Ruvic acceda a tu base de datos en modo solo lectura: puede consultar información, pero nunca modificarla. Una vez configurado, puedes pedirle al agente cosas como:

  • “Genera un reporte con las ventas del último trimestre y envíalo por correo.”
  • “¿Cuántos clientes nuevos se registraron esta semana?”
  • “Revisa por qué este query está lento y proponme mejoras.”

Operaciones soportadas:

The SQL Server connector lets the Ruvic agent access your database in read-only mode: it can query information but never modify it. Once configured, you can ask the agent things like:

  • “Build a report with last quarter's sales and email it.”
  • “How many new customers signed up this week?”
  • “Check why this query is slow and suggest improvements.”

Supported operations:

OperaciónOperation Qué haceWhat it does
db.list_tables Lista las tablas de la base de datos con su cantidad de filasLists the database tables with their row counts
db.read Lee los datos de una tabla (con límite de filas)Reads data from a table (with a row limit)
db.stats Estadísticas de una tabla: filas, tamaño, columnas y tiposTable statistics: rows, size, columns and types
db.explain Analiza el plan ESTIMADO de ejecución de una consulta sin ejecutarlaAnalyzes a query's ESTIMATED execution plan without running it
Solo lectura, por diseñoRead-only by design El login dedicado del conector solo tiene permiso SELECT (y SHOWPLAN para analizar queries). Aunque el agente lo intentara, el servidor rechazaría cualquier escritura. The connector's dedicated login only has SELECT permission (and SHOWPLAN to analyze queries). Even if the agent tried, the server would reject any write.

2. Prerrequisitos2. Prerequisites

  • Una instancia de Microsoft SQL Server 2016 o superior (on-premise, Azure SQL Database/Managed Instance, AWS RDS for SQL Server, o Docker).
  • Acceso administrativo a la instancia (rol sysadmin o equivalente), solo durante la configuración inicial.
  • Autenticación en modo mixto habilitada (SQL Server and Windows Authentication mode) — el conector usa usuario/contraseña, no autenticación de Windows.
  • Acceso de red desde la plataforma Ruvic al servidor (host y puerto alcanzables; regla de firewall si aplica).
  • Los datos de conexión: host, puerto (por defecto 1433) y nombre de la base de datos.
  • A Microsoft SQL Server 2016+ instance (on-premise, Azure SQL Database/Managed Instance, AWS RDS for SQL Server, or Docker).
  • Administrative access to the instance (sysadmin role or equivalent), only during initial setup.
  • Mixed mode authentication enabled (SQL Server and Windows Authentication mode) — the connector uses username/password, not Windows authentication.
  • Network access from the Ruvic platform to the server (reachable host and port; firewall rule if applicable).
  • The connection details: host, port (default 1433) and database name.

3. Configuración en el fabricante (paso a paso)3. Vendor-side setup (step by step)

Vas a crear un login dedicado de solo lectura para Ruvic. No uses el login sa ni el de tu aplicación: un login dedicado permite auditar la actividad del agente y limitar el acceso al mínimo.

You will create a dedicated read-only login for Ruvic. Do not reuse the sa login or an application login: a dedicated login lets you audit the agent's activity and keep access to a minimum.

  1. Habilita la autenticación en modo mixto (si no lo está).

    En SQL Server Management Studio (SSMS): clic derecho sobre el servidor → Properties → Security → selecciona SQL Server and Windows Authentication mode → Aceptar → reinicia el servicio de SQL Server.

    Enable mixed mode authentication (if not already).

    In SQL Server Management Studio (SSMS): right-click the server → Properties → Security → select SQL Server and Windows Authentication mode → OK → restart the SQL Server service.

    [ Captura: pantalla de Security en Server Properties con el modo mixto seleccionado — reemplazar por assets/img/mssql-mixed-mode.png ] [ Screenshot: Security page in Server Properties with mixed mode selected — replace with assets/img/mssql-mixed-mode.png ]
  2. Conéctate a la instancia como administrador.

    Usa SSMS, Azure Data Studio o sqlcmd con un login administrador:

    Connect to the instance as an administrator.

    Use SSMS, Azure Data Studio or sqlcmd with an admin login:

    sqlcmd -S TU_HOST,1433 -U sa -P TU_CONTRASEÑA_ADMIN
  3. Crea el login y el usuario de solo lectura para Ruvic.

    Elige una contraseña fuerte (mínimo 16 caracteres, generada aleatoriamente; SQL Server exige que cumpla la política de complejidad) y guárdala: la usarás en el formulario de Ruvic (campo Contraseña).

    Create the read-only login and user for Ruvic.

    Choose a strong password (16+ random characters; SQL Server enforces its complexity policy) and save it: you will paste it into the Ruvic form (Password field).

    CREATE LOGIN ruvic_reader WITH PASSWORD = 'CAMBIA_ESTA_CONTRASEÑA';

USE tu_base_de_datos;
CREATE USER ruvic_reader FOR LOGIN ruvic_reader;
  4. Otorga los permisos de lectura y de plan de ejecución.

    Sobre el esquema que quieras exponer al agente (el ejemplo usa dbo; repite para otros esquemas):

    Grant read and execution-plan permissions.

    On the schema you want to expose to the agent (the example uses dbo; repeat for other schemas):

    GRANT SELECT ON SCHEMA::dbo TO ruvic_reader;
GRANT SHOWPLAN TO ruvic_reader;
    NotaNote GRANT SELECT ON SCHEMA::dbo aplica automáticamente a las tablas que se creen después en ese esquema, sin repetir el GRANT. SHOWPLAN permite ver el plan de un query sin ejecutarlo ni dar permiso de escritura. GRANT SELECT ON SCHEMA::dbo automatically covers tables created afterwards in that schema, without re-running the GRANT. SHOWPLAN allows viewing a query's plan without executing it or granting write access.
  5. Habilita el protocolo TCP/IP y el acceso de red desde Ruvic.

    Muchas instalaciones traen TCP/IP deshabilitado por defecto:

    • Servidor propio: abre SQL Server Configuration Manager → SQL Server Network Configuration → Protocols for [tu instancia] → habilita TCP/IP → reinicia el servicio. Luego permite el puerto 1433 en el firewall de Windows para la IP de Ruvic.
    • Azure SQL / AWS RDS: agrega la IP de la plataforma Ruvic en el firewall de Azure SQL o en el security group de RDS.
    Enable the TCP/IP protocol and network access from Ruvic.

    Many installations ship with TCP/IP disabled by default:

    • Self-hosted: open SQL Server Configuration Manager → SQL Server Network Configuration → Protocols for [your instance] → enable TCP/IP → restart the service. Then allow port 1433 in the Windows firewall for the Ruvic IP.
    • Azure SQL / AWS RDS: add the Ruvic platform IP to the Azure SQL firewall or the RDS security group.
    [ Captura: SQL Server Configuration Manager con TCP/IP habilitado — reemplazar por assets/img/mssql-tcpip.png ] [ Screenshot: SQL Server Configuration Manager with TCP/IP enabled — replace with assets/img/mssql-tcpip.png ]
  6. Verifica el login desde fuera del servidor.

    Desde una máquina distinta al servidor de BD (idealmente con salida de red similar a la de Ruvic):

    Verify the login from outside the server.

    From a machine other than the DB server (ideally with network egress similar to Ruvic's):

    sqlcmd -S TU_HOST,1433 -U ruvic_reader -P TU_CONTRASEÑA -Q "SELECT 1"

    Si responde 1, el fabricante quedó listo. Anota estos valores para la siguiente sección: host, puerto, base de datos, usuario y contraseña.

    If it returns 1, the vendor side is ready. Write down these values for the next section: host, port, database, user and password.

4. Permisos requeridos4. Required permissions

Principio de mínimo privilegio: el login de Ruvic solo necesita estos permisos. No otorgues roles sysadmin, db_owner, de escritura ni de administración.

Least-privilege principle: the Ruvic login only needs these permissions. Do not grant sysadmin, db_owner, write or admin roles.

PermisoPermission Para qué lo usa el conectorWhy the connector needs it
SELECT (por esquema)(per schema) Leer datos (db.read), contar filas y calcular estadísticas (db.stats)Read data (db.read), count rows and compute statistics (db.stats)
SHOWPLAN Compilar y ver el plan estimado de un query sin ejecutarlo (db.explain)Compile and view a query's estimated plan without executing it (db.explain)

Las vistas de catálogo que usa el conector (sys.tables, sys.columns, sys.partitions, sys.allocation_units) son visibles para cualquier login con algún permiso sobre el objeto correspondiente; no requieren VIEW SERVER STATE ni permisos adicionales.

The catalog views the connector uses (sys.tables, sys.columns, sys.partitions, sys.allocation_units) are visible to any login with some permission on the corresponding object; they don't require VIEW SERVER STATE or extra permissions.

6. Verificación6. Verification

  1. Test de conexión exitoso.

    El botón Probar conexión debe responder: “Conexión exitosa a tu-host:1433/tu_base”. La tarjeta del conector cambia a estado Configurado.

    Successful connection test.

    The Test connection button should return: “Successful connection to your-host:1433/your_db”. The connector card switches to Configured.

  2. Automatización mínima de prueba.

    Abre una conversación nueva con el agente y pídele:

    “Lista las tablas de la base de datos SQL Server y dime cuántas filas tiene cada una.”

    El agente debe responder con el listado real de tus tablas. Si lo hace, el conector quedó operativo de punta a punta.

    Minimal test automation.

    Open a new conversation with the agent and ask:

    “List the tables in the SQL Server database and tell me how many rows each one has.”

    The agent should reply with your actual table list. If it does, the connector is fully operational.

8. Preguntas frecuentes y notas de seguridad8. FAQ and security notes

¿El agente puede modificar o borrar datos?Can the agent modify or delete data?

No. El login ruvic_reader solo tiene permiso SELECT y SHOWPLAN. El conector nunca ejecuta sentencias de escritura, y aunque lo intentara, el servidor la rechazaría por permisos.

No. The ruvic_reader login only has SELECT and SHOWPLAN permission. The connector never issues write statements, and even if it tried, the server would reject it due to permissions.

¿Dónde queda guardada la contraseña?Where is the password stored?

Como secreto en el almacén de credenciales de la plataforma, asociado a tu usuario. La interfaz nunca vuelve a mostrarla; solo indica “definida”. Para cambiarla, escribe una nueva en el formulario; para conservarla al editar otros campos, deja el campo vacío.

As a secret in the platform's credential store, tied to your user. The UI never displays it again; it only shows “set”. To change it, type a new one in the form; to keep it while editing other fields, leave the field empty.

Rotación de credencialesCredential rotation

Recomendado cada 90 días o según tu política: cambia la contraseña en SQL Server (ALTER LOGIN ruvic_reader WITH PASSWORD = '...';), actualízala en el formulario del conector y pulsa Probar conexión.

Recommended every 90 days or per your policy: change the password in SQL Server (ALTER LOGIN ruvic_reader WITH PASSWORD = '...';), update it in the connector form and click Test connection.

¿Qué hago si la contraseña se compromete?What if the password is compromised?

  1. Cambia la contraseña en SQL Server de inmediato (ALTER LOGIN ... WITH PASSWORD = ...) o deshabilita el login (ALTER LOGIN ruvic_reader DISABLE;).
  2. Revisa la actividad reciente del login en los logs del servidor / Extended Events.
  3. Actualiza la nueva contraseña en el formulario del conector.
  1. Change the password in SQL Server immediately (ALTER LOGIN ... WITH PASSWORD = ...) or disable the login (ALTER LOGIN ruvic_reader DISABLE;).
  2. Review the login's recent activity in the server logs / Extended Events.
  3. Update the new password in the connector form.

¿Expiran las credenciales?Do the credentials expire?

Por defecto no, salvo que el login se cree con CHECK_EXPIRATION = ON siguiendo la política de contraseñas de Windows. Si tu política exige expiración, actívala al crear o alterar el login y programa la rotación antes de esa fecha.

By default no, unless the login is created with CHECK_EXPIRATION = ON following the Windows password policy. If your policy requires expiration, enable it when creating or altering the login and schedule rotation before that date.

Buenas prácticasBest practices Usa siempre un login dedicado y de solo lectura; restringe el acceso de red al mínimo necesario; no expongas esquemas con datos sensibles que el agente no necesite; y evita compartir la contraseña por canales no seguros. Always use a dedicated read-only login; restrict network access to the minimum necessary; do not expose schemas with sensitive data the agent doesn't need; and never share the password over insecure channels.
