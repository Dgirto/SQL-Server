¿Qué es y para qué sirve?

El conector SQL Server permite que el agente de Ruvic acceda a tu base de datos en modo solo lectura: puede consultar información, pero nunca modificarla. Una vez configurado, puedes pedirle al agente cosas como:

  • “Genera un reporte con las ventas del último trimestre y envíalo por correo.”
  • “¿Cuántos clientes nuevos se registraron esta semana?”
  • “Revisa por qué este query está lento y proponme mejoras.”

Operaciones soportadas:

Operación Qué hace
db.list_tables Lista las tablas de la base de datos con su cantidad de filas
db.read Lee los datos de una tabla (con límite de filas)
db.stats Estadísticas de una tabla: filas, tamaño, columnas y tipos
db.explain Analiza el plan ESTIMADO de ejecución de una consulta sin ejecutarla
Solo lectura, por diseño El login dedicado del conector solo tiene permiso SELECT (y SHOWPLAN para analizar queries). Aunque el agente lo intentara, el servidor rechazaría cualquier escritura.

Prerrequisitos

  • Una instancia de Microsoft SQL Server 2016 o superior (on-premise, Azure SQL Database/Managed Instance, AWS RDS for SQL Server, o Docker).
  • Acceso administrativo a la instancia (rol sysadmin o equivalente), solo durante la configuración inicial.
  • Autenticación en modo mixto habilitada (SQL Server and Windows Authentication mode) — el conector usa usuario/contraseña, no autenticación de Windows.
  • Acceso de red desde la plataforma Ruvic al servidor (host y puerto alcanzables; regla de firewall si aplica).
  • Los datos de conexión: host, puerto (por defecto 1433) y nombre de la base de datos.

Configuración en el fabricante

Vas a crear un login dedicado de solo lectura para Ruvic. No uses el login sa ni el de tu aplicación: un login dedicado permite auditar la actividad del agente y limitar el acceso al mínimo.

  1. Habilita la autenticación en modo mixto (si no lo está).

    En SQL Server Management Studio (SSMS): clic derecho sobre el servidor → Properties → Security → selecciona SQL Server and Windows Authentication mode → Aceptar → reinicia el servicio de SQL Server.

    [ Captura: pantalla de Security en Server Properties con el modo mixto seleccionado — reemplazar por assets/img/mssql-mixed-mode.png ]
  2. Conéctate a la instancia como administrador.

    Usa SSMS, Azure Data Studio o sqlcmd con un login administrador:

    sqlcmd -S TU_HOST,1433 -U sa -P TU_CONTRASEÑA_ADMIN
  3. Crea el login y el usuario de solo lectura para Ruvic.

    Elige una contraseña fuerte (mínimo 16 caracteres, generada aleatoriamente; SQL Server exige que cumpla la política de complejidad) y guárdala: la usarás en el formulario de Ruvic (campo Contraseña).

    CREATE LOGIN ruvic_reader WITH PASSWORD = 'CAMBIA_ESTA_CONTRASEÑA';

USE tu_base_de_datos;
CREATE USER ruvic_reader FOR LOGIN ruvic_reader;
  4. Otorga los permisos de lectura y de plan de ejecución.

    Sobre el esquema que quieras exponer al agente (el ejemplo usa dbo; repite para otros esquemas):

    GRANT SELECT ON SCHEMA::dbo TO ruvic_reader;
GRANT SHOWPLAN TO ruvic_reader;
    Nota GRANT SELECT ON SCHEMA::dbo aplica automáticamente a las tablas que se creen después en ese esquema, sin repetir el GRANT. SHOWPLAN permite ver el plan de un query sin ejecutarlo ni dar permiso de escritura.
  5. Habilita el protocolo TCP/IP y el acceso de red desde Ruvic.

    Muchas instalaciones traen TCP/IP deshabilitado por defecto:

    • Servidor propio: abre SQL Server Configuration Manager → SQL Server Network Configuration → Protocols for [tu instancia] → habilita TCP/IP → reinicia el servicio. Luego permite el puerto 1433 en el firewall de Windows para la IP de Ruvic.
    • Azure SQL / AWS RDS: agrega la IP de la plataforma Ruvic en el firewall de Azure SQL o en el security group de RDS.
    [ Captura: SQL Server Configuration Manager con TCP/IP habilitado — reemplazar por assets/img/mssql-tcpip.png ]
  6. Verifica el login desde fuera del servidor.

    Desde una máquina distinta al servidor de BD (idealmente con salida de red similar a la de Ruvic):

    sqlcmd -S TU_HOST,1433 -U ruvic_reader -P TU_CONTRASEÑA -Q "SELECT 1"

    Si responde 1, el fabricante quedó listo. Anota estos valores para la siguiente sección: host, puerto, base de datos, usuario y contraseña.

Permisos requeridos

Principio de mínimo privilegio: el login de Ruvic solo necesita estos permisos. No otorgues roles sysadmin, db_owner, de escritura ni de administración.

Permiso Para qué sirve
SELECT (por esquema) Leer datos (db.read), contar filas y calcular estadísticas (db.stats)
SHOWPLAN Compilar y ver el plan estimado de un query sin ejecutarlo (db.explain)

Las vistas de catálogo que usa el conector (sys.tables, sys.columns, sys.partitions, sys.allocation_units) son visibles para cualquier login con algún permiso sobre el objeto correspondiente; no requieren VIEW SERVER STATE ni permisos adicionales.

Configuración en Ruvic

Entra a Settings → Conectores, busca la tarjeta SQL Server y pulsa Configurar. Diligencia el formulario con los valores que obtuviste en la sección anterior:

Campo del formulario Valor
Host Hostname o IP del servidor (ej. db.empresa.com)
Puerto 1433 (salvo configuración distinta)
Base de datos Nombre de la base de datos a consultar
Usuario ruvic_reader
Contraseña La contraseña del paso 3. Se guarda cifrada como secreto: la interfaz no volverá a mostrarla
Timeout de conexión Opcional; 10 segundos por defecto

Guarda y pulsa Probar conexión — la plataforma se conectará a tu base de datos y confirmará que las credenciales y la red funcionan.

[ Captura: formulario del conector SQL Server en Settings → Conectores — reemplazar por assets/img/mssql-form.png ]

Verificación

  1. Test de conexión exitoso.

    El botón Probar conexión debe responder: “Conexión exitosa a tu-host:1433/tu_base”. La tarjeta del conector cambia a estado Configurado.

  2. Automatización mínima de prueba.

    Abre una conversación nueva con el agente y pídele: “Lista las tablas de la base de datos SQL Server y dime cuántas filas tiene cada una.” El agente debe responder con el listado real de tus tablas. Si lo hace, el conector quedó operativo de punta a punta.

Solución de problemas

Mensaje / síntoma Causa probable y solución
“Autenticación fallida: usuario o contraseña inválidos” Contraseña mal copiada, login inexistente, o la autenticación en modo mixto no está habilitada (revisa el paso 1 de la sección 3). Reintenta y vuelve a guardar el formulario (deja la contraseña vacía para conservar la anterior o escribe la nueva para reemplazarla).
“No se pudo conectar … (timeout)” La plataforma no alcanza el host/puerto. Revisa el paso 5 (TCP/IP habilitado, firewall, security group) y que el host no sea localhost (debe ser accesible desde Ruvic, no desde tu equipo).
“La base de datos 'X' no existe” Nombre de base de datos mal escrito en el formulario, o el usuario no fue creado en esa base (recuerda: CREATE USER se ejecuta dentro de USE tu_base_de_datos;). Verifica con SELECT name FROM sys.databases;.
El agente ve las tablas pero “permission was denied” al leer una Falta el GRANT SELECT sobre ese esquema. Repite el paso 4 de la sección 3.
“The user does not have permission to perform this action” al analizar un query Falta GRANT SHOWPLAN. Repite el paso 4 de la sección 3.
El conector aparece “Sin configurar” tras guardar Algún campo obligatorio quedó vacío. Ábrelo de nuevo y completa host, puerto, base de datos, usuario y contraseña.

Preguntas frecuentes y notas de seguridad

¿El agente puede modificar o borrar datos?

No. El login ruvic_reader solo tiene permiso SELECT y SHOWPLAN. El conector nunca ejecuta sentencias de escritura, y aunque lo intentara, el servidor la rechazaría por permisos.

¿Dónde queda guardada la contraseña?

Como secreto en el almacén de credenciales de la plataforma, asociado a tu usuario. La interfaz nunca vuelve a mostrarla; solo indica “definida”. Para cambiarla, escribe una nueva en el formulario; para conservarla al editar otros campos, deja el campo vacío.

Rotación de credenciales

Recomendado cada 90 días o según tu política: cambia la contraseña en SQL Server (ALTER LOGIN ruvic_reader WITH PASSWORD = '...';), actualízala en el formulario del conector y pulsa Probar conexión.

¿Qué hago si la contraseña se compromete?

  1. Cambia la contraseña en SQL Server de inmediato (ALTER LOGIN ... WITH PASSWORD = ...) o deshabilita el login (ALTER LOGIN ruvic_reader DISABLE;).
  2. Revisa la actividad reciente del login en los logs del servidor / Extended Events.
  3. Actualiza la nueva contraseña en el formulario del conector.

¿Expiran las credenciales?

Por defecto no, salvo que el login se cree con CHECK_EXPIRATION = ON siguiendo la política de contraseñas de Windows. Si tu política exige expiración, actívala al crear o alterar el login y programa la rotación antes de esa fecha.

Buenas prácticas Usa siempre un login dedicado y de solo lectura; restringe el acceso de red al mínimo necesario; no expongas esquemas con datos sensibles que el agente no necesite; y evita compartir la contraseña por canales no seguros.
